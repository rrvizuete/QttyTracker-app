# UI_GUIDE.md

## Goal
The app should have a professional, modern construction-management look inspired by the screenshots in `/docs/ui-reference`.

## General Style
- Clean, polished, enterprise-style UI
- Modern but restrained
- Professional rather than playful
- Spacious layout with clear hierarchy
- Rounded cards and panels
- Subtle shadows
- Strong alignment and consistent spacing

## Layout
- Left sidebar navigation
- Top header bar with project selector and user menu
- Main content area with cards, tables, and charts
- Dashboard pages should feel structured and calm, not crowded

## Colors
- Neutral base palette
- White or very light gray backgrounds
- Dark gray text
- One primary accent color for actions and highlights
- Use muted status colors for success, warning, error

## Components
### Cards
- Rounded corners
- Light shadow
- Clear title and optional subtitle
- Comfortable padding

### Buttons
- Primary buttons should feel solid and modern
- Secondary buttons should be quieter
- Destructive actions should be clearly distinct

### Tables
- Highly readable
- Subtle row separators
- Good spacing
- Sticky headers if practical
- Important numeric columns right-aligned
- Contain Hierarchies and nested entries

### Forms
- Clean labels
- Good spacing between fields
- Strong focus states
- Validation messages should be clear and inline

### Dashboards
- Use summary KPI cards at top
- Use clean chart containers
- Keep visual density moderate
- Prioritize readability over decoration

## Notes
- Use the reference screenshots as inspiration, not as exact copies
- Maintain consistency across all pages
- Favor reusable components
